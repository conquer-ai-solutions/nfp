Architecture Proposal

What if editing a label couldn't break the math?

Plan Pulse does critical work. But calculations and presentation are wired together today. One small change can cascade. Here's how we fix that.

Demo client: Congreso de Latinos Unidos
⚠
The problem today

Math and templates are tangled

Calculations live inside the same code that renders the report. That means:

  • Renaming a label can silently break a formula
  • Updating a formula risks breaking the layout
  • Swapping in a new client's data means touching multiple files
  • No clean data layer to run QC checks against before anything renders
✓
The fix

Separate data, math, and display

Three layers, each doing one job. Nothing else:

  • Layer 1 — Data: One JSON file per client. Single source of truth. Drop in new data, everything recalculates
  • Layer 2 — Calculations: Pure math functions. Loss ratios, funding models, Monte Carlo. Isolated from HTML
  • Layer 3 — Templates: Renders results to screen. Edit any label, move any element. The math can't see it

How the three layers connect

Data flows in one direction. Each layer only talks to the next.
LAYER 1

Client Data

Structured JSON extracted from intake documents. Census, claims, plan details. One file per client.

LAYER 2

Calculation Engine

Pure functions that take data in and return results. No HTML, no formatting, no side effects.

LAYER 3

Report Templates

Takes calculated results and renders them. Labels, layout, styling. Safe to edit. Nothing breaks.

Under the Hood

This is what the separation looks like in practice. Layer 1 is data. Layer 2 is math. Layer 3 is display. They never touch each other.

Swap this file for a new client. Every module, every calculation, every chart regenerates automatically.
f(x) Layer 2 — Calc Engine
FUNCTIONS
// Pure functions. Data in, results out. // They never touch the DOM or HTML. function calcLossRatio(claims) { return claims.total_claims_paid / claims.total_premium * 100; } function calcFunding(claims, assumptions) { const fi = claims.total_premium; const sf = claims.total_claims_paid * (1 + assumptions.trend) + assumptions.admin * members * 12 + assumptions.stopLoss + assumptions.margin; return { fi, sf, savings: fi - sf }; }
Rename any label in the report. These functions keep running identically. That's the whole point.
■ Layer 3 — Template Renderer
PRESENTATION
// The template ONLY renders. It calls calc functions and displays results. // It never contains formulas or business logic. function renderFundingTable() { // Call the calc engine const result = calcFunding(clientData.claims, getAssumptions()); // Just display — no math here document.getElementById('fi-total').textContent = fmt(result.fi); document.getElementById('sf-total').textContent = fmt(result.sf); document.getElementById('savings').textContent = fmt(result.savings); } // Safe to rename any label, restyle any element, reorder any column. // The math doesn't know. The math doesn't care.
This is why the team can safely edit reports without fear of breaking calculations. Edit the presentation, not the logic.
