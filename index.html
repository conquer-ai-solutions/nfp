The four issues you described
Each one maps to the same root cause: math and templates aren't separated.
ISSUE 1

Small edits cascade

You change "PPO" to "Preferred Provider Organization" and the system rewrites more than just that label. The edit touches things it shouldn't.

"Sometimes it changes a lot more than what we're intending to, so we have a little bit of an architecture issue."
ISSUE 2

Hard-coded vs dynamic is unclear

On the dynamic modules like the funding analysis, sometimes hitting Calculate updates everything. Sometimes it doesn't. You can't tell which values are hard-coded and which are live.

"We've struggled with figuring out which buttons are hard-coded versus which buttons are not hard-coded."
ISSUE 3

Data carries between analyses

When running a new analysis, data from the previous one sometimes carries over. The system doesn't cleanly reset between runs.

"Sometimes it seems to carry one analysis, the data from one analysis into the next."
ISSUE 4

The advanced modules are fragile

The simple data-display modules work fine. But the dynamic ones — funding analysis, Monte Carlo scenario modeling — are where things break. More math means more risk of something going wrong.

"Some of these are a little bit more advanced than others. And obviously those are the ones that we're struggling with."
Our recommendation: one structural change fixes all four
Separate the data, the math, and the display into three independent layers. Each one does one job.
LAYER 1

Client Data

One structured file per client. Census, claims, benefit details. Extracted from intake docs, validated by QC. Swapping this file is how you switch analyses — nothing else changes.

Fixes: data carrying between analyses
LAYER 2

Calculation Engine

Loss ratios, funding comparisons, Monte Carlo — all pure math functions. They read data in, return results out. They never see HTML. There's no ambiguity about what's hard-coded.

Fixes: hard-coded confusion, fragile dynamic modules
LAYER 3

Report Templates

HTML modules that display calculated results. Change any label, rearrange any section, restyle anything. The math is in Layer 2 and can't be affected by what happens here.

Fixes: small edits cascading

Under the Hood

This is what the three-layer separation actually looks like. Layer 1 is just data. Layer 2 is just math. Layer 3 is just display. They never mix.

FUNCTIONS
// Pure functions. Data in, results out. // They never see HTML. They never see labels. function calcLossRatio(claims) { return claims.total_claims_paid / claims.total_premium * 100; } function calcFunding(claims, assumptions) { const fi = claims.total_premium; const sf = claims.total_claims_paid * (1 + assumptions.trend) + assumptions.admin * members * 12 + assumptions.stopLoss + assumptions.margin; return { fi, sf, savings: fi - sf }; } // Every value is explicit. // Nothing is hard-coded in a template.
